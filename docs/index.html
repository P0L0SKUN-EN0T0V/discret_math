Формализмы моделирования дискретных систем

Конечные автоматы, сети Петри и другие инструменты для описания поведения систем

О чём это

Формализм моделирования — это математический язык для описания поведения системы. Вместо того чтобы объяснять словами «когда нажали кнопку — включается мотор, а когда отпустили — выключается», мы рисуем точную модель с состояниями, переходами и правилами.

Зачем: модель можно анализировать (найти тупики, гонки, недостижимые состояния), верифицировать (доказать что система не зависнет) и автоматически реализовать (сгенерировать код).

Какие формализмы существуют

Каждый формализм заточен под свой класс задач. Вот основные — от простых к сложным:

Конечные автоматы (FSM)

базовый

Состояния + переходы по событиям. Самый простой и понятный формализм. Основа всех остальных.

Сети Петри

параллелизм

Места + переходы + токены. Умеют моделировать параллельные процессы, синхронизацию, ресурсы.

Statecharts (Харел)

иерархия

FSM на стероидах: вложенные состояния, параллельные регионы, история. Стандарт UML.

Временные сети Петри

время

Сети Петри с задержками. Анализ производительности, тайм-ауты, real-time системы.

Деревья поведения

робототехника

Дерево из задач с приоритетами. Модульность и переиспользование. Стандарт в робототехнике и геймдеве.

GRAFCET / SFC

промышленность

Стандарт IEC 61131-3 для ПЛК. Шаги + переходы + параллельные ветви. Язык заводской автоматики.

Другие формализмы

Цепи Маркова, сети массового обслуживания, алгебры процессов (CSP, CCS), клеточные автоматы, гибридные автоматы — краткий обзор и когда их применять.
