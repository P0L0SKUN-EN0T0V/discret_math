Временные сети Петри

Timed Petri Nets — моделирование задержек, тайм-аутов и производительности

Зачем нужно время

Классические сети Петри описывают порядок событий, но не длительность. Вопросы «сколько деталей в час обработает конвейер?» или «успеет ли робот закончить до тайм-аута?» они не ответят.

Временные сети Петри добавляют задержки к переходам или местам, превращая логическую модель в количественную. Теперь можно считать пропускную способность, среднее время цикла, утилизацию ресурсов.

Три подхода к добавлению времени

Timed Petri Nets (Рамчандани, 1974)

Каждому переходу приписана детерминированная задержка $d_j \geq 0$. После разрешения переход «горит» $d_j$ единиц времени, затем срабатывает.

Формально: $N = (P, T, F, W, M_0, D)$, где $D: T \rightarrow \mathbb{R}_{\geq 0}$

Time Petri Nets (Мерлен, 1976)

Каждому переходу приписан интервал $[a_j, b_j]$. Переход ДОЛЖЕН сработать не раньше $a_j$ и не позже $b_j$ после разрешения.

Строже: моделирует обязательные дедлайны. Если $b_j$ истёк, а переход не сработал — это ошибка модели.

Stochastic Petri Nets (SPN)

Задержки — случайные величины с экспоненциальным распределением: $d_j \sim \text{Exp}(\lambda_j)$. Это позволяет использовать аппарат марковских цепей для анализа.

Расширения: GSPN (Generalized SPN) добавляют мгновенные переходы (без задержки) наряду с временными.

Семантика срабатывания

Для Timed Petri Nets (фиксированная задержка):

  1. Переход $t_j$ разрешается при маркировке $M$ (как в обычной сети)
  2. Токены во входных местах резервируются (другие переходы их не видят)
  3. Через $d_j$ единиц времени переход срабатывает: забирает зарезервированные, кладёт в выходные

Важный нюанс: если токены резервируются (semantics: eager) — они недоступны другим переходам во время ожидания. Если не резервируются (race) — возможна гонка: два перехода спорят за один токен, побеждает тот, чья задержка короче.

Пример: конвейерная линия

Три станка последовательно: фрезерный (8с), сверлильный (5с), шлифовальный (10с). Между станками — буферы на 3 детали.

Места: $p_1$ = вход (∞ деталей), $p_2$ = буфер 1 (≤3), $p_3$ = буфер 2 (≤3), $p_4$ = выход

Переходы: $t_1$ = фреза ($d=8$с), $t_2$ = сверло ($d=5$с), $t_3$ = шлифовка ($d=10$с)

Анализ:

  • Узкое место = шлифовка (10с) — она определяет максимальную пропускную способность: $\frac{1}{10}$ = 6 деталей/мин
  • Буфер перед шлифовкой будет заполняться (сверло быстрее), пока не достигнет лимита 3 — тогда сверло начнёт простаивать
  • Утилизация фрезы: $\frac{8}{10} = 80\%$, сверла: $\frac{5}{10} = 50\%$, шлифовки: $100\%$

Методы анализа

Симуляция

Прогоняем модель N раз, собираем статистику. Просто, но нужно много прогонов для точности.

Инструменты: CPN Tools, PIPE, GreatSPN

Марковский анализ (для SPN)

Строим CTMC (Continuous Time Markov Chain) из графа достижимости. Решаем систему $\pi Q = 0$ для стационарного распределения.

Точный, но число состояний может быть огромным.

Max-plus алгебра (для TPN)

Для детерминированных задержек: заменяем (×, +) на (max, +). Система становится линейной в max-plus алгебре, цикл находится как собственное значение.

Элегантный подход для конвейеров и расписаний.

Формулы Литтла

$L = \lambda W$ — среднее число элементов в системе = интенсивность × среднее время. Связывает метрики без полного анализа.

Работает для систем в стационарном режиме.

Применение в real-time системах

Time Petri Nets (с интервалами) особенно важны для real-time систем, где нарушение дедлайна = катастрофа:

  • Авионика: ответ системы управления полётом в пределах [0, 20ms]
  • Автопилот: обработка данных лидара за [5ms, 50ms]
  • Медицина: кардиостимулятор — интервал между импульсами [800ms, 1200ms]

Верификация: инструмент UPPAAL проверяет свойства вида «переход ВСЕГДА срабатывает до дедлайна» с помощью model checking на временных автоматах (близкий формализм).

Дальше

Временные сети Петри хороши для анализа производительности, но не самый удобный инструмент для описания поведения агентов. Переходим к деревьям поведения (Behaviour Trees) — формализму из робототехники и геймдева.
