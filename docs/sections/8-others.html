Другие формализмы

Цепи Маркова, алгебры процессов, клеточные автоматы, гибридные автоматы и другие

Помимо основных шести формализмов (FSM, сети Петри, Statecharts, TPN, BT, GRAFCET) существует ещё ряд важных моделей. Каждая заточена под свой класс задач.

Цепи Маркова

Дискретная цепь Маркова (DTMC) — конечный автомат, где переходы вероятностные: из состояния $i$ в состояние $j$ с вероятностью $p_{ij}$, причём $\sum_j p_{ij} = 1$.

$$P(X_{n+1} = j \mid X_n = i) = p_{ij}$$

Ключевое свойство (марковское): будущее зависит только от настоящего, не от прошлого. «Не помнит, как попал в текущее состояние.»

Что считают:

  • Стационарное распределение $\pi$: $\pi P = \pi$ — в каком состоянии система будет «в среднем»
  • Среднее время возврата, вероятность поглощения
  • MTTF (Mean Time To Failure) — надёжность

Применение: анализ надёжности, PageRank (Google), биоинформатика (HMM), финансы, телеком (модели трафика).

CTMC — Continuous Time Markov Chain

Непрерывный аналог: время в каждом состоянии — экспоненциальная случайная величина. Матрица переходов заменяется на матрицу интенсивностей $Q$.

Связь с сетями Петри: стохастические сети Петри (SPN) при анализе сводятся к CTMC.

Сети массового обслуживания

Queueing Networks — формализм для систем с очередями: запросы приходят, ждут в очереди, обслуживаются, уходят. Основа теории массового обслуживания (ТМО).

Нотация Кендалла: $A/S/c/K$ — тип входного потока / распределение обслуживания / число серверов / размер буфера.

  • $M/M/1$ — пуассоновский поток, экспоненциальное обслуживание, 1 сервер
  • $M/D/c$ — пуассоновский поток, детерминированное обслуживание, $c$ серверов

Применение:

  • Проектирование серверов: сколько инстансов нужно для 99-го перцентиля < 200ms?
  • Производственные линии: оптимальный размер буфера между станками
  • Телеком: сколько каналов нужно для заданного процента отказов

Связь с сетями Петри: места = очереди, переходы = серверы, токены = запросы.

Алгебры процессов

Текстовые (алгебраические) формализмы для описания параллельных и взаимодействующих процессов. Процессы определяются рекурсивными уравнениями с операторами.

CSP (Communicating Sequential Processes)

Хоар, 1978. Процессы обмениваются через синхронные каналы. Оба процесса должны быть готовы для передачи.

BUFF = left?x → right!x → BUFF
SYS = PROD ∥ BUFF ∥ CONS

Повлиял на Go (goroutines + channels).

CCS (Calculus of Communicating Systems)

Милнер, 1980. Процессы взаимодействуют через комплементарные действия: $a$ и $\bar{a}$ синхронизируются в $\tau$ (внутреннее действие).

P = a.P' + b.P''
Q = ā.Q'
P | Q → P' | Q' (синхронизация)

Основа π-calculus (мобильные процессы).

Применение: верификация протоколов (Bluetooth, TLS), модель параллельных программ, формальная спецификация.

Инструменты: FDR4 (для CSP), CADP (для CCS/LOTOS), mCRL2.

Клеточные автоматы

Cellular Automata (CA) — решётка клеток, каждая в одном из конечного числа состояний. На каждом шаге состояние клетки обновляется по локальному правилу, зависящему от соседей.

$$s_i^{t+1} = f(s_{i-1}^t, s_i^t, s_{i+1}^t)$$

Примеры

  • Game of Life (Конвей) — 2D решётка, жив/мёртв, правила 23/3
  • Rule 110 — 1D, доказана Тьюринг-полнота
  • Lattice Boltzmann — моделирование гидродинамики

Применение

  • Моделирование трафика (Nagel-Schreckenberg)
  • Распространение огня, эпидемий
  • Самоорганизация, эмерджентное поведение
  • Криптография (CA-based PRNG)

Гибридные автоматы

Hybrid Automata — объединяют дискретное и непрерывное: конечный автомат + дифференциальные уравнения в каждом состоянии. Состояние системы = (дискретный режим, непрерывный вектор).

$$H = (Q, X, \textit{Init}, f, \textit{Inv}, E, G, R)$$

  • $Q$ — дискретные режимы, $X \subseteq \mathbb{R}^n$ — непрерывные переменные
  • $f: Q \times X \rightarrow \mathbb{R}^n$ — динамика ($\dot{x} = f(q, x)$) в каждом режиме
  • $\textit{Inv}: Q \rightarrow 2^X$ — инвариант (пока $x \in \textit{Inv}(q)$, остаёмся в $q$)
  • $G$ — guard (условие переключения), $R$ — reset (сброс переменных при переходе)

Пример: термостат. Два режима: «нагрев» ($\dot{T} = +k_1$) и «охлаждение» ($\dot{T} = -k_2$). Переключение: нагрев → охлаждение при $T \geq T_{max}$, охлаждение → нагрев при $T \leq T_{min}$.

Применение: автопилоты, робототехника (контакт/бесконтакт), электроника (DC-DC конвертеры), биология (генные сети).

Инструменты: UPPAAL, SpaceEx, HyTech, PHAVer.

Сложность: верификация гибридных автоматов в общем случае неразрешима. Даже для простых линейных систем задача достижимости неразрешима. Существуют разрешимые подклассы: timed automata (только часы), rectangular automata.

Временные автоматы (Timed Automata)

Подкласс гибридных автоматов, где непрерывные переменные — только часы ($\dot{x} = 1$). Достаточно выразительны для real-time систем, но задача верификации разрешима (PSPACE-complete).

Стандартный инструмент: UPPAAL — model checker для сетей временных автоматов. Верификация свойств в TCTL (Timed CTL).

Близки к Time Petri Nets по выразительности, но с другим синтаксисом и инструментарием.
