GRAFCET / SFC

Стандарт IEC 61131-3 для программирования промышленных контроллеров

Что это

GRAFCET (GRAphe Fonctionnel de Commande Étape-Transition) — графический язык, разработанный во Франции в 1977 году для описания последовательного управления в промышленности. Стандартизован как IEC 60848.

SFC (Sequential Function Chart) — его программная реализация в стандарте IEC 61131-3 для ПЛК (программируемых логических контроллеров). По сути это GRAFCET, адаптированный для прямого выполнения на контроллере.

GRAFCET — это специализированный потомок сетей Петри, заточенный под промышленную автоматику. Из сетей Петри он унаследовал параллельные ветви и синхронизацию, но упростил и регламентировал всё остальное.

Элементы GRAFCET

Шаг (étape / step)

Прямоугольник с номером. Может быть активным или неактивным. К активному шагу привязаны действия, которые выполняются.

Начальный шаг выделяется двойной рамкой.

Переход (transition)

Горизонтальная чёрточка между шагами. Переход срабатывает, когда: (1) все предшествующие шаги активны И (2) условие перехода истинно.

Условие — логическое выражение от входов контроллера.

Действие (action)

Привязано к шагу. Типы: N (пока шаг активен), S (set — установить), R (reset — сбросить), P (импульс при активации), D (с задержкой), L (ограниченное по времени).

Параллельные ветви

Двойная горизонтальная линия = fork (разветвление) или join (синхронизация). Все ветви между fork и join выполняются параллельно.

Пример: автоматический пресс

S0: Ожидание
── кнопка «Старт» ──
S1: Подача детали
── датчик «деталь на месте» ──
S2: Прессование
── таймер 5с истёк ──
S3: Подъём пресса
── датчик «пресс вверху» ──
S4: Выгрузка
── деталь убрана ──
↩ возврат в S0

Действия, привязанные к шагам:

ШагДействиеТип
S0Индикатор «Готов» = ONN (пока активен)
S1Конвейер подачи = ONN
S2Цилиндр пресса = DOWN; Запустить таймер 5сN; D(5s)
S3Цилиндр пресса = UPN
S4Конвейер выгрузки = ONN

Параллельные ветви

GRAFCET поддерживает одновременное выполнение нескольких ветвей — это прямое наследие сетей Петри. При срабатывании перехода перед fork все ветви активируются одновременно. Join ждёт завершения ВСЕХ ветвей.

S0: Старт
════════════════ (fork)
 S1: Нагреть      S2: Подать газ
 S3: Ждать 60°C   S4: Ждать давление
════════════════ (join)
S5: Начать процесс

S5 активируется только когда И температура достигнута, И давление достигнуто

Ограничения GRAFCET гарантируют, что все свойства (достижимость, живость, ограниченность) проверяются эффективно.

SFC в стандарте IEC 61131-3

IEC 61131-3 определяет пять языков программирования ПЛК:

  1. SFC — Sequential Function Chart (= GRAFCET) — структура программы
  2. LD — Ladder Diagram — релейные схемы
  3. FBD — Function Block Diagram — функциональные блоки
  4. ST — Structured Text — текстовый (похож на Pascal)
  5. IL — Instruction List — ассемблер (устарел)

SFC — это «верхний уровень», а действия внутри шагов пишутся на любом из остальных четырёх языков. Типичная комбинация: SFC для логики + ST для вычислений.

Инструменты

CODESYS

Бесплатная IDE для программирования ПЛК. Полная поддержка IEC 61131-3, включая SFC. Симулятор.

Siemens TIA Portal

Промышленный стандарт для Siemens S7. GRAPH (аналог SFC) + LAD + SCL.

OpenPLC

Open-source ПЛК на Raspberry Pi / Arduino. Поддержка SFC через OpenPLC Editor.

Beremiz

Open-source IDE (Python). Все 5 языков IEC 61131-3, экспорт в C-код.
