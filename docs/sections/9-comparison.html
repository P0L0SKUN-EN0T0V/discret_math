Дерево решений: какой формализм выбрать

Задача?
├─ Последовательная логика, ≤15 состояний → FSM
├─ Параллельные процессы + разделяемые ресурсы → Сети Петри
├─ Сложная иерархия режимов (UML, встраиваемые) → Statecharts
├─ Анализ производительности, пропускной способности → TPN / SPN
├─ Поведение робота / NPC (модульное, расширяемое) → BT
├─ Программирование ПЛК, промышленная автоматика → GRAFCET / SFC
├─ Вероятностный анализ, надёжность → Цепи Маркова
├─ Верификация протоколов, формальные доказательства → CSP / CCS
├─ Дискретно-непрерывные системы (hybrid) → Гибридные автоматы
└─ Real-time с верификацией дедлайнов → Timed Automata (UPPAAL)

По областям применения

Робототехника

  1. BT — поведение, mission planning
  2. FSM / Statecharts — режимы работы
  3. Гибридные автоматы — контакт/бесконтакт

Промышленная автоматика

  1. GRAFCET/SFC — основной язык
  2. Сети Петри — анализ безопасности
  3. TPN — оптимизация циклов

Встраиваемые системы

  1. Statecharts — Stateflow, кодогенерация
  2. FSM — простые контроллеры
  3. Timed Automata — верификация

IT / Серверная инфраструктура

  1. Queueing Networks — capacity planning
  2. Цепи Маркова — SLA, надёжность
  3. Сети Петри — workflow, BPMN

Иерархия выразительности

Машина Тьюринга — всё вычислимое
⊃ строго мощнее
Сети Петри + ингибиторные дуги = Тьюринг-полные
⊃
Сети Петри (классические) — строго между
⊃
FSM / Statecharts / BT / GRAFCET — регулярные языки

Statecharts и BT эквивалентны FSM по выразительности (те же регулярные языки), но экспоненциально компактнее в записи.

Практические рекомендации

  1. Начинай с FSM. Если хватает — не усложняй. FSM покрывает 80% задач.
  2. Перешёл 15 состояний? Подумай о Statecharts (иерархия) или BT (модульность).
  3. Есть параллелизм + ресурсы? Сети Петри — единственный формализм, который это делает естественно.
  4. Нужна производительность? TPN или SPN для количественного анализа.
  5. Промышленный контроллер? GRAFCET/SFC — стандарт, который понимает любой автоматчик.
  6. Комбинируй. BT для верхнего уровня + FSM для отдельных поведений — стандарт в робототехнике.

Итог

Формализмы моделирования — это не академическая абстракция, а рабочие инструменты инженера. Каждый решает свой класс задач:

  • FSM — простая последовательная логика
  • Сети Петри — параллелизм и ресурсы
  • Statecharts — сложная иерархия режимов
  • TPN/SPN — анализ производительности
  • BT — модульное поведение агентов
  • GRAFCET — промышленная автоматика

Выбор формализма — это как выбор языка программирования: нет «лучшего», есть подходящий для задачи. Главное — знать, что существует, и уметь выбрать.
